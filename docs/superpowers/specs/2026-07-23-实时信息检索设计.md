# 实时信息检索设计

## 目标

当用户询问最新新闻、时事动态或带有时间范围的信息时，必须先执行真实网络搜索；模型仅整理本次搜索结果，不得使用训练数据补全实时事实。

## 范围

第一期继续复用项目现有的 `BaiduSearchService`。不改动图片、天气、翻译和普通文本对话链路。

## 查询流程

1. 后端先识别实时信息请求，并解析消息中的时间表达式。
2. 将相对或绝对时间统一为上海时区下的起止日期。例如：昨天、最近一个月、今年 5 到 7 月。
3. 后端直接调用百度搜索，不依赖模型决定是否调用，也不允许模型生成第二次检索词。
4. 搜索成功后，使用不带聊天记忆、也不再提供工具的专用模型调用整理结果。
5. 专用提示要求模型仅依据搜索结果作答，保留来源链接；没有明确发布时间的结果必须提示以原文为准。
6. 搜索失败或无结果时，后端直接返回失败或无结果提示，不调用模型生成兜底内容。

## 时间表达式

- 昨天：当天减一天的单日范围。
- 最近 N 天或最近一个月：当天向前推算的闭区间。
- 今年 M 月到 N 月：当前年份对应的月度范围；结束月为当前月时，结束日为当天。
- 明确年份、月份或日期：按用户给出的范围解析。
- 未指定范围但包含最新、新闻、实时或动态：以当前日期作为查询上下文，不宣称严格发布时间筛选。

## 数据源边界

现有百度搜索返回标题、摘要和链接，未提供标准化发布时间。因此第一期可保证每次请求已真实搜索，但不能严格验证每条结果确实落在指定日期范围。回答必须保留链接，并对没有明确发布时间的内容标注“发布时间以原文为准”。

## 异常处理

- 搜索服务异常：返回“暂未取得实时搜索结果，请稍后重试”。
- 搜索无结果：返回“未检索到该主题与时间范围内的实时结果”。
- 不允许将失败信息作为普通工具结果交给模型继续自由回答。

## 验收与验证

- “今天最新新闻”必定执行一次搜索，且模型不会进行第二次搜索。
- “昨天的新闻”“最近一个月的新闻”“今年 5 到 7 月的新闻”均生成正确的查询时间条件。
- 搜索失败时不调用模型，回复不包含训练数据兜底措辞。
- 最终回答仅引用本次搜索结果，含来源链接。
